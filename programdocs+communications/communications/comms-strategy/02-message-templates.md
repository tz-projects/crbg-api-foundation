# Message Templates (skeletons)

> Structure, not full prose. Fill in per phase. Keep every message: why-led, new-vs-existing explicit, hub-linked, support-pointed.

## Common message shape (all phases)
- **Subject** — concrete, includes the program name.
- **From / endorsement** — sponsor named where it matters (Phase 0/1/4).
- **Audience** — who this is for.
- **Opening — the why** — one or two sentences, outcome-framed.
- **Body — the spine** — the substance for this phase.
- **New vs existing block** — what applies to each population.
- **The ask** — the specific action + date.
- **Links** — relevant hub docs.
- **Support** — channel + office hours.
- **Sign-off.**

## Phase 0 — Pre-brief (note)
Briefing deck / note, not an email blast. Spine: what's coming, why now, timeline, the ask of leaders, what you need from them.

## Phase 1 — Announcement
Spine: introduce program + standard; the why; what governance is/isn't; the model in brief; the full timeline; reassurance (no blocking yet). New-vs-existing stated plainly.

## Phase 2 — Visibility (per-team)
Spine: report-only is on, nothing blocks; here's *your* report; how to check locally; how to read results; where the fix guide is.

## Phase 3 — Remediation
Spine: how to fix at scale (playbook); office hours; progress so far; recognition; optional carrot.

## Phase 4 — Warning
Spine: dated gate notice; countdown reminders; waiver path; new APIs already gated; what to do now.

## Phase 5 — Live
Spine: gate is on; soft/hard behaviour; what a block looks like; waiver path; support.
