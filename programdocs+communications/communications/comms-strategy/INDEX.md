# API Governance — Communications Strategy (Index)

> INTERNAL change-management playbook for the platform team. NOT team-facing. Lives in a separate internal/run space, not in the governance hub the 85 teams read.

## Files
1. `00-comms-strategy-overview.md` — framing, audiences, channels, cadence, content principles
2. `01-phase-sequence.md` — the six communication phases, each with the same sub-structure
3. `02-message-templates.md` — skeleton templates per phase (structure, not full prose)

## Doc ↔ comms alignment (the progressive reveal)
Each phase surfaces the documents that become relevant then. Fundamentals first, operational detail as needed.

| Phase | What it socializes | Hub docs it links |
|---|---|---|
| 0 Pre-brief (internal) | The plan, the ask of leaders | Overview, Charter, Principles, Timeline |
| 1 Announcement / why + model | The fundamentals | Overview, Glossary, How Governance Works, Principles, Charter, Timeline |
| 2 Visibility / the mirror | Where you stand, how to check | Rules Catalogue, Validation Guide, Definition of Done, Per-Team Report |
| 3 Supported remediation | How to fix, where help is | Top-Violations Playbook, Support, FAQ |
| 4 Enforcement warning | The date, the escape hatch | Timeline, Waiver Process |
| 5 Enforcement live | Gate on, what to do if blocked | Waiver Process, Support, Timeline |

## How to use
Work top to bottom. Don't advance a phase until its success signal (in `01-phase-sequence.md`) is met.
