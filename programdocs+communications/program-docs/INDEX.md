# API Governance — Confluence Hub (Index)

This folder mirrors the Confluence page tree one-to-one. Each `.md` file = one page.

## Structure
- `foundational/` — what the program is and how to think about it (read first)
- `operational/` — what to actually do

## Page order
**Foundational**
1. Program Overview (Start Here / front door)
2. Key Concepts & Glossary
3. How Governance Works Here (the model)
4. Guiding Principles
5. Governance Charter (authority capstone)

**Operational**
6. Governance Rules Catalogue
7. Self-Service Validation Guide
8. Definition of Done
9. Enforcement Timeline — What Happens When
10. Exception / Waiver Process
11. Top-Violations Remediation Playbook *(populate after report-only)*
12. FAQ
13. Per-Team Current-State Report *(generated; data model parked)*
14. Support & Help

## Pushing to Confluence
1. Create the parent space, then two parent pages: **Foundational** and **Operational**.
2. Create child pages in the order above.
3. Paste each file's contents into its page (Confluence Cloud auto-converts Markdown on paste).
4. Set the Program Overview page as the space landing page — it's the front door and links everywhere else.
5. Every external email links to the space root, never to attachments.
