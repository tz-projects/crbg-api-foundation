# API Governance Charter

> The authority capstone — the only foundational page that carries a mandate and a sponsor signature. Different tone from the Overview: formal, invocable.

## Purpose & mandate
What authority this program operates under and what it is empowered to require.

## Scope
In scope (which APIs, which platforms) / out of scope.

## The standard adopted
OAS + the custom rule set, by reference to the Rules Catalogue.

## Policy statements
The binding statements, stated separately for new vs existing APIs.

## Roles & responsibilities
Platform team, API owners, approvers, sponsor.

## Exceptions
Pointer to the Waiver Process; states that waivers are the only sanctioned escape from the gate.

## Sponsorship & sign-off
Named executive sponsor + sign-off block.

## Review cadence
How often the charter and ruleset are reviewed.
