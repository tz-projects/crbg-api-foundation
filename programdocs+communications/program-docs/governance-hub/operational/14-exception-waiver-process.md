# Exception / Waiver Process

> The pressure-release valve that lets the gate stay on. Without it, the first legitimate block triggers an escalation that switches the gate off.

## What a waiver is (and isn't)
A sanctioned, time-bound exemption — not a permanent pass.

## What qualifies
Deprecated, frozen, sunsetting, ownerless, or genuinely untouchable APIs; criteria.

## How to request one
The request path + required information.

## Who approves
Approver role + decision criteria.

## Duration & expiry
Default lifetime, renewal, what happens at expiry.

## What the owner must acknowledge
The risk/conditions the requester signs up to.

## What a waived API still must do
Any residual obligations (e.g. register, no new endpoints).
